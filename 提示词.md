# 一、项目目标

开发一个 Windows 10/11 桌面运行的 2D 动漫桌宠。

桌宠角色来源于用户已经提供的真人照片。需要先分析真人照片，制作能够保留人物主要识别特征的动漫化角色，再将动漫角色制作成透明背景的桌宠动作素材，最后由桌宠程序播放这些动画并提供桌面互动。

完整流程：

```text
真人参考照片
→ 人物特征分析
→ 动漫化角色设定
→ 角色一致性规范
→ 桌宠动作素材
→ 透明 PNG 序列帧
→ 桌宠动画系统
→ Windows 桌面交互
→ EXE 打包
```

项目第一版不加入：

- AI 对话；
- 大语言模型接口；
- 语音识别；
- 语音合成；
- 登录注册；
- 云端同步；
- 数据库；
- 社交功能；
- 复杂的 Live2D 建模。

第一版的重点是：

1. 真人特征保留；
2. 动漫角色前后一致；
3. 动画自然可爱；
4. 桌宠运行稳定；
5. 素材可以替换；
6. 项目可以打包成 EXE。

# 四、动漫角色风格

桌宠采用：

```text
日系二维动画风格
Q版但不过度幼化
2.5～3.5头身
清晰赛璐璐上色
轮廓干净
色彩柔和
轻微可爱化
保留真人辨识度
适合小尺寸桌面显示
透明背景
```

避免：

- 过度幼儿化；
- 五官与真人完全无关；
- 极度夸张的大眼睛；
- 写实照片风格；
- 3D塑料模型感；
- 厚重油画感；
- 复杂背景；
- 模糊边缘；
- 多余文字；
- 水印；
- 随机改变发型；
- 随机改变服装；
- 随机改变饰品；
- 不同图片中角色年龄感明显变化。

默认制作全身角色，因为桌宠需要行走、坐下、睡觉和拖拽动画。

# 七、动作规划

第一版至少包含以下动作：

## 1. idle

待机循环：

- 自然站立；
- 呼吸起伏；
- 偶尔眨眼；
- 头发轻微摆动；
- 身体轻微晃动。

推荐 6～12 帧。

## 2. walk

左右行走：

- 使用完整步态循环；
- 脚底不能明显漂移；
- 重心变化自然；
- 默认只制作一个方向；
- 程序通过水平镜像支持另一个方向。

推荐 8～12 帧。

## 3. sit

从站立到坐下：

- 站立；
- 屈膝；
- 下蹲；
- 坐稳；
- 保持坐姿。

推荐 6～10 帧。

## 4. sleep

睡觉循环：

- 坐下或趴下；
- 闭眼；
- 身体轻微呼吸；
- 可出现简洁的 Z 字符号，但不要影响透明背景。

推荐 6～10 帧。

## 5. wave

挥手：

- 抬手；
- 挥动；
- 微笑；
- 放下手。

推荐 6～10 帧。

## 6. happy

开心互动：

- 眼睛弯起；
- 身体轻轻跳起；
- 手部做开心动作。

推荐 6～10 帧。

## 7. surprised

受到单击后的惊讶动作：

- 身体轻微后仰；
- 眼睛睁大；
- 短暂恢复。

推荐 4～8 帧。

## 8. dragged

被鼠标拖动：

- 双脚离地；
- 身体轻微下坠；
- 表情有些惊讶；
- 保持可循环。

推荐 4～8 帧。

## 9. fall

可选动作：

- 从空中下落；
- 接触桌面；
- 轻微反弹；
- 重新站稳。

- 图片尺寸；

# 十、程序核心功能

## 1. 透明桌宠窗口

必须实现：

- 无边框；
- 背景透明；
- 始终置顶；
- 不显示普通任务栏按钮；
- 桌面上只显示人物；
- 默认位于屏幕右下角；
- 保持在任务栏上方；
- 使用 availableGeometry；
- 不得走出可用屏幕区域；
- 支持 Windows 缩放比例；
- 基础多显示器兼容。

## 2. PNG 序列帧系统

实现独立动画播放器：

- 加载 PNG 序列；
- 自动排序；
- 支持 FPS；
- 支持循环；
- 支持单次播放；
- 支持结束回调；
- 支持水平镜像；
- 支持缓存；
- 支持预加载；
- 动作缺失时回退；
- 避免反复读取磁盘造成卡顿。

## 3. 状态机

至少包括：

```text
IDLE
WALK_LEFT
WALK_RIGHT
SIT
SLEEP
WAVE
HAPPY
SURPRISED
DRAGGED
FALL
```

状态机与窗口显示逻辑必须分离。

状态规则：

- 启动进入 IDLE；
- 空闲后随机选择行为；
- 行走时根据方向镜像；
- 坐下后可能进入睡觉；
- 点击睡觉中的角色时醒来；
- 拖动时进入 DRAGGED；
- 松开后根据位置进入 FALL 或 IDLE；
- 双击进入 HAPPY；
- 单次动画结束后返回合理状态；
- 不允许状态之间发生明显冲突。

## 4. 自动行为

支持：

- 自动待机；
- 随机行走；
- 随机坐下；
- 随机睡觉；
- 偶尔挥手；
- 到达屏幕边缘转向；
- 用户可暂停自动行为；
- 行走速度与概率可配置；
- 行为切换不能过于频繁。

## 5. 鼠标交互

实现：

- 左键拖拽；
- 单击反馈；
- 双击反馈；
- 右键菜单；
- 区分单击与拖拽；
- 防止拖出屏幕；
- 拖动后记录位置；
- 点击透明区域时尽量不误触。

右键菜单：

```text
挥手
坐下
睡觉
随机动作
暂停自动行为
恢复自动行为
重置位置
设置
关于
退出
```

## 6. 系统托盘

托盘功能：

```text
显示桌宠
隐藏桌宠
暂停
恢复
重置位置
设置
关于
退出
```

关闭主窗口时隐藏到托盘，只有选择退出才结束进程。

## 7. 设置窗口

至少支持：

- 开启或关闭自动移动；
- 调整角色缩放；
- 调整移动速度；

- 保存设置；
