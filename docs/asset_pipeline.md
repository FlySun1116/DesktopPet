## 制作步骤

1. **授权与盘点**：确认肖像权和用途；选取清晰主参考，其他照片仅补足角度、发型、服装信息。
2. **可见特征分析**：记录脸型、发型轮廓、刘海、发色、眼镜/饰品和服装；不做身份或敏感属性推断。
3. **角色设定**：先生成 front、左右 45°、side、back、expressions、palette 以及总设定图。正式动作生成必须引用通过审核的设定图。
4. **动作规划**：按 idle、walk、sit、sleep、wave、happy、surprised、dragged、fall 分别制作。全身、768×768、底部中心锚点，编号 `001.png` 起连续递增。
5. **透明处理**：输出真实 RGBA，透明区 alpha=0；清除隐藏 RGB、白边和色键污染。不允许棋盘格或白/黑背景伪装透明。
6. **自动检查**：运行 `python scripts/validate_assets.py`，检查动作缺失、帧数、编号、尺寸、alpha、空帧、边缘、边界与脚底漂移。
7. **视觉检查**：在深/浅棋盘背景分别查看；审核裁切、内部透明裂缝、身份漂移、服装变化、锚点跳动和动作连续性。
8. **运行验收**：生成动作预览并在真实桌面逐项播放，验证镜像、缩放和高 DPI。

## 正式与占位的判定

只有同时满足以下条件才可标为正式：有授权；参考来源记录完整；人物一致性人工通过；所有帧真实透明且验证报告通过；动作预览通过；没有未处理的生成水印或背景。任何程序绘制的几何人物、色块、简笔图或未做人像复核的生成图都必须标记 `PLACEHOLDER`。

## 角色包合同

每个角色目录包含 `character.json`、可选 `preview.png` 和动作子目录。`character.json` 声明画布、锚点、默认缩放和每个动作的相对文件夹、FPS、循环方式。缺失非核心动作可在运行时回退 `idle`，但发布验收必须列出缺失项。

## 推荐人工阈值

- 同一动作所有帧尺寸完全一致；画布推荐 768×768。
- alpha 同时包含 0 与非 0 像素；角色不得触碰画布边缘。
- 脚底中位数变化应小，跳跃/拖拽/下落按动作豁免但仍需视觉合理。
- idle 6–12 帧、walk 8–12 帧，其余遵循风格指南；编号无缺口。
